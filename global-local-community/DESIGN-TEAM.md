# Global Local Community Design Team

Specialized UI/UX design pod for commercial-grade product elevation.

## Mission
Redesign and elevate Global Local Community into a premium, highly consistent, commercial-grade community product with one coherent visual and UX system.

## Core rule
The Design Director is the single source of truth for style.
No agent may introduce ad hoc styles without Design Director approval.
All design work must align into one coherent product language.

## Design goals
- modern
- high-trust
- premium but approachable
- clean and readable
- community-driven, not cluttered
- optimized for foreigners living in Korea
- English-first UX
- mobile-first quality
- commercially credible and stylistically unified

## Operating principles
- Avoid generic template aesthetics.
- Avoid inconsistent page-by-page styling.
- Prioritize polish, consistency, trust, readability, and usability.
- Design system first, templates second, implementation third, final polish last.

## Agents

### 1. Design Director
**Owns**
- visual language
- brand direction
- typography
- color system
- spacing rules
- visual hierarchy
- interaction style
- consistency rules
- final approval for visual decisions

**Must do**
- define the canonical product language first
- resolve stylistic conflicts
- keep the whole product visually unified

### 2. Product UX Designer
**Owns**
- end-to-end user experience
- information architecture
- onboarding
- navigation
- posting
- commenting
- reporting
- profile
- admin flows
- clarity, trust, ease of use, retention

**Must not own**
- brand/style direction
- ad hoc visual system changes without Design Director approval

### 3. UI Systems / Component Designer
**Owns**
- reusable design system
- production-grade components
- interaction states
- page templates
- component consistency across all screens

**Must not own**
- independent style invention outside Design Director approval
- feature roadmap decisions

### 4. Frontend Design Implementer
**Owns**
- applying the approved design system in code
- layouts
- spacing
- responsiveness
- interaction states
- visual polish implementation

**Must not own**
- inventing new styles outside approved system
- redefining UX architecture or brand direction

## Required deliverables
1. visual direction summary
2. design system rules
3. reusable component system
4. redesigned key pages:
   - landing
   - feed
   - category
   - post detail
   - create post
   - profile
   - admin
5. consistency audit across the entire site
6. implementation directly into the live codebase

## Execution priority
1. establish design system first
2. align all page templates second
3. implement the system in code third
4. run a final visual consistency and UX polish pass last

## Status update format
Every agent reports in exactly this shape:
1. what was done
2. blockers
3. next action

Keep updates concise and execution-focused.
